Tetragon Runtime Security

eBPF-based kernel-level security observability — detect attacker behaviour inside any container, in real time.

WHAT IS TETRAGON?

Tetragon is the runtime security layer built into this cluster alongside Cilium. It uses eBPF to hook directly into the Linux kernel — observing system calls inside every running container, with near-zero overhead and no code changes required.

By default Tetragon is passive — it can see everything but alerts only on what you configure via TracingPolicies. When a policy is deployed, Tetragon matches specific kernel calls and emits structured real-time security events, visible in the live stream below.

Process Execution

Detect unexpected binaries inside pods — shells, curl, wget, nmap.

File Access

Alert on reads of /etc/shadow, service account tokens, and sensitive paths.

Kill & Enforce

Optionally terminate offending processes in real time — not just observe.
