Overview

A self-contained Kubernetes demo environment showcasing Cisco's Zero Trust Access portfolio — fully integrated and ready to run.

WHAT IS POC IN A POD?

PoC in a Pod is a portable, self-contained demo environment running on a single Ubuntu VM. It bundles Cisco's key security products — all pre-configured and integrated — so you can demonstrate, explore, and test a complete Zero Trust Access story without complex infrastructure setup.

Everything runs as Kubernetes pods: protected demo workloads, the Secure Access Resource Connector, network enforcement, runtime security, and observability tooling. The PoC Dashboard ties it all together with guided configuration for each module.

WHAT IS ZERO TRUST ACCESS?

Traditional VPN grants users broad network access the moment they authenticate. That means a single compromised account can reach everything. Zero Trust Access (ZTA) flips this model — every connection is verified based on user identity, device health, and context before access is granted. Users reach only the specific application they need, nothing more.

VERIFY IDENTITY

Every access request is authenticated — user identity, device posture, and risk level are checked before any connection is established.

LEAST-PRIVILEGE ACCESS

Users reach specific applications, not the entire network. Policies define exactly who can access which resource, on which port, from which context.

Verified User → Verified Application Port
CONTINUOUS ENFORCEMENT

Network and runtime security layers enforce policies in real time — microsegmentation, process monitoring, and behavioral threat detection keep enforcing trust after access is granted.
