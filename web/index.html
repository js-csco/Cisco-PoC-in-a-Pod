WHY THE IP TELLS THE STORY

With Cisco Secure Access ZTNA your browser never opens a direct connection to this server. Traffic flows through the Secure Access cloud gateway and then through the on-premises Connector pod running in this k3s cluster — so the server sees the Connector's internal cluster IP, not yours.

✅ Via ZTNA Server sees the Connector's private cluster IP. Your real IP is never exposed to the application.
⚠️ Direct Access Server sees your real public IP. Zero Trust policies are not in effect for this connection.
