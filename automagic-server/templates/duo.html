DUO POLICIES

Configure Duo enrollment and authentication policies for the PoC environment. These settings must be applied manually in the Duo Admin Panel.

1. Enrollment Policy

In the Duo Admin Panel, navigate to Policies → User Enrollment Policies and apply the following settings:

  1. Policy Enablement: Check the box next to "Enable the enrollment and account management policies on this page".
  2. Under Global Enrollment Policy, configure:
    • Require Password: enable the checkbox.
    • Authenticators: allow only the following:
      • Passkey (Current Device)
      • Passkey (Mobile Device) or Security Key
      • Duo Mobile

2. Global Policy

In the Duo Admin Panel, navigate to Policies → Policies and edit the Global Policy:

  1. New User Policy: set to "Require enrollment" so all users must complete MFA setup.
  2. Authentication Methods: only recommended authenticators (Platform Authenticator, Roaming Authenticator, Duo Push) for both second factor and passwordless SSO.
  3. Risk-based Factor Selection: disabled — don’t limit authentication methods based on risk.
  4. User Location: leave as default ("No restriction").

USER & GROUP CREATION

Add up to three users for your Duo PoC environment. At least one user is required. Click "Add Users and Groups" to create the users, the "PoC Users" group, and assign everyone to it.

INTEGRATIONS

1. Cisco Secure Access

In the Duo Admin Panel, go to Applications → Protect an Application, search for Cisco Secure Access, and follow the setup guide. This integration covers two parts:

a) SAML Integration

Configure Single Sign-On between Duo and Cisco Secure Access. Follow the SSO setup guide to connect Duo as the identity provider.

Duo SSO – Cisco Secure Access Docs

b) User Provisioning

Enable SCIM provisioning in the same integration to sync Duo users and groups to Cisco Secure Access.

Tip — Provisioning tab → Groups section: In the Select Groups field, make sure to select PoC Users so that only your PoC users are synced to Cisco Secure Access.

2. Generic SAML Application

Creates a Generic SAML Service Provider integration in Duo called "PoC in a Pod: SAML App" and assigns it to the PoC Users group.

This will:
  • Create a sso-generic integration via the Duo Admin API
  • Restrict access to the PoC Users group
  • Return the integration key and metadata needed for SAML configuration

IDENTITY INTELLIGENCE

Identify, reduce, and respond to identity-based threats.

  1. Go to Monitoring → Cisco Identity Intelligence and click "Connect to Cisco Identity Intelligence".
  2. Click on "Set Up SSO Access".
  3. On the Application Page, click on the app called cii-sso-… and add the PoC Users group to "Enable only for permitted groups" under the User Access section.
  4. Go back to Monitoring → Cisco Identity Intelligence and click "Launch Identity Intelligence".
