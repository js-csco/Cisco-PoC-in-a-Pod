DUO POLICIES

Configure Duo enrollment and authentication policies to enforce strong, phishing-resistant MFA across your PoC environment. These settings must be applied manually in the Duo Admin Panel.

1. Enrollment Policy

In the Duo Admin Panel, navigate to Policies → User Enrollment Policies and apply the following settings:

  1. Policy Enablement: Check the box next to "Enable the enrollment and account management policies on this page".
  2. Under Global Enrollment Policy, configure:
    • Require Password: enable the checkbox.
    • Authenticators: allow only the following:
      • Passkey (Current Device)
      • Passkey (Mobile Device) or Security Key
      • Duo Mobile

2. Global Policy — Authentication Methods

In the Duo Admin Panel, navigate to Policies → Policies and edit the Global Policy:

  1. Under Authentication Methods, select only "Recommended Authenticators for Second Factor and Passwordless in SSO". This allows all recommended authenticators while excluding weaker methods.

3. Duo Passport

In the Duo Admin Panel, navigate to Policies → Passport and click "Set up Passport". Select "On for all users" as recommended.

USER & GROUP CREATION

Add up to three users for your Duo PoC environment. At least one user is required. Click "Add Users and Groups" to create the users, the "PoC Users" group, and assign everyone to it.

Add Users and Groups will perform the following:
  • Create each provided user with email and username
  • Create "PoC Users" group (if it doesn't exist)
  • Add all users to "PoC Users" group
  • Send enrollment email to each new user
Next step after creating users: Go to the Duo Admin Dashboard → Users and share the enrollment link and code with the user. Enrollment works best in an incognito window.

INTEGRATIONS

A single integration connects Duo with Cisco Secure Access, covering both SSO and user provisioning (SCIM). In the Duo Admin Panel, go to Applications → Protect an Application, search for Cisco Secure Access, and follow the setup guide to configure SSO and enable user provisioning.

Tip — Provisioning tab → Groups section: In the Select Groups field, make sure to select PoC Users so that only your PoC users are synced to Cisco Secure Access.

IDENTITY INTELLIGENCE

Identify, reduce, and respond to identity-based threats.

  1. Go to Monitoring → Cisco Identity Intelligence and click "Connect to Cisco Identity Intelligence".
  2. Click on "Set Up SSO Access".
  3. On the Application Page, click on the app called cii-sso-… and add the PoC Users group to "Enable only for permitted groups" under the User Access section.
  4. Go back to Monitoring → Cisco Identity Intelligence and click "Launch Identity Intelligence".
