Overview

A self-contained Kubernetes demo environment showcasing Cisco's Zero Trust Application Access portfolio — fully integrated and ready to run.

WHAT IS POC IN A POD?

PoC in a Pod is a portable, self-contained demo environment running on a single node using k3s (lightweight Kubernetes). It bundles Cisco's key security products — all pre-configured and integrated — so you can demonstrate, explore, and test the full Zero Trust Application Access story without complex infrastructure setup.

Everything runs as Kubernetes pods: the protected demo application, the ZTNA connector, network enforcement, runtime security, and observability tooling. This Automagic Server ties it all together with guided configuration steps for each module.

WHAT IS ZERO TRUST APPLICATION ACCESS?

Traditional VPN grants users broad network access once authenticated. Zero Trust Application Access (ZTAA) replaces this with identity-centric, per-application access that continuously verifies trust — before and during every session. Users never touch the network directly; they reach specific applications only when policy permits.

VERIFY IDENTITY

Every access request is authenticated — user identity, device posture, and risk level are evaluated before any connection is established.

LEAST-PRIVILEGE ACCESS

Users reach specific applications, not the entire network. Policies define who can access what resource, on which port, from which context.

CONTINUOUS ENFORCEMENT

Network and runtime security layers enforce policies in real time — microsegmentation, process monitoring, and behavioral threat detection.
