Overview

A self-contained Kubernetes demo environment showcasing Cisco's Zero Trust Application Access portfolio — fully integrated and ready to run.

WHAT IS POC IN A POD?

PoC in a Pod is a portable, self-contained demo environment running on a single node using k3s (lightweight Kubernetes). It bundles Cisco's key security products — all pre-configured and integrated — so you can demonstrate, explore, and test the full Zero Trust Application Access story without complex infrastructure setup.

Everything runs as Kubernetes pods: the protected demo application, the ZTNA connector, network enforcement, runtime security, and observability tooling. This Automagic Server ties it all together with guided configuration steps for each module.

WHAT IS ZERO TRUST APPLICATION ACCESS?

Traditional VPN grants users broad network access once authenticated. Zero Trust Application Access (ZTAA) replaces this with identity-centric, per-application access that continuously verifies trust — before and during every session. Users never touch the network directly; they reach specific applications only when policy permits.

VERIFY IDENTITY

Every access request is authenticated — user identity, device posture, and risk level are evaluated before any connection is established.

LEAST-PRIVILEGE ACCESS

Users reach specific applications, not the entire network. Policies define who can access what resource, on which port, from which context.

CONTINUOUS ENFORCEMENT

Network and runtime security layers enforce policies in real time — microsegmentation, process monitoring, and behavioral threat detection.

COMPONENTS
Cisco Secure Access
Cloud · ZTNA Gateway

Provides Zero Trust Network Access. The on-prem Connector in the cluster tunnels traffic from the cloud gateway to private resources — no inbound firewall rules needed.

Duo
Cloud · Identity & MFA

Multi-factor authentication and Identity Intelligence. Provides passwordless MFA, device trust verification, and risk-based authentication policies integrated with Secure Access.

Cilium
k3s DaemonSet · CNI / Network Policy

eBPF-based CNI providing Layer 4/7 network policies and microsegmentation. Enforces which pods can communicate and on which ports — without iptables overhead.

Tetragon
k3s DaemonSet · Runtime Security

eBPF-based runtime security. Monitors process execution, file access, network calls, and syscall activity across all pods in real time. Events are exported to Splunk.

Splunk
Pod · Port 30500 · SIEM

Log aggregation and SIEM. Ingests security events from Tetragon, Cilium, and application logs for full visibility, dashboards, and alerting.

ThousandEyes
Cloud · Network Monitoring

End-to-end network path visibility. Monitors connectivity and performance for the ZTNA access path from user to application.

SSE Check
Pod · Port 30550 · Demo App

A demo web application serving as the private resource behind Zero Trust. Accessible only through Cisco Secure Access policies — not directly from the internet.

RDP / SSH Servers
Pods · Port 30389 / 30022

Remote desktop and SSH servers exposed as private resources. Demonstrates Cisco Secure Access protecting non-web protocols via browser-based agentless access.

Dashy
Pod · Port 30100 · Dashboard

Self-hosted link dashboard. Provides a single start page with tiles linking to all services in the PoC environment.

PoC Playbook
Pod · Port 30250 · Test Cases

Guided test case runner for validating the full PoC — covering Duo, Secure Access, Cilium, and Tetragon with pass/fail tracking.
