LOG SOURCES

Splunk receives logs from these Cisco services via the Cisco Security Cloud app. Configure each source in the Cisco admin portal to forward to this Splunk instance.

Cisco Duo & Identity Intelligence
Authentication logs, admin actions, telephony events, and trust assessments from Duo MFA and Identity Intelligence risk scoring. Configure the Duo Admin API integration in the Cisco Security Cloud app with your API hostname, integration key, and secret key from the Duo Admin Panel.
Duo Admin API docs
Cisco Secure Access
DNS, web proxy, firewall, and DLP logs from Cisco Secure Access (formerly Umbrella). Configure the Secure Access Reporting API integration in the Cisco Security Cloud app with your API key and key secret from the Secure Access dashboard.
Secure Access reporting docs

MCP SERVER CONNECTION

Once the Splunk MCP Server app is installed, AI agents can connect to Splunk over the Model Context Protocol (MCP) to run searches, explore knowledge objects, and query data.

CONNECT YOUR MCP CLIENT
Endpoint: https://{{ server_ip }}:30502/services/mcp
Transport: Streamable HTTP (SSE)  ·  Auth: Splunk credentials (admin / piap-admin)
Example — Claude Desktop / Claude Code config
{
  "mcpServers": {
    "splunk": {
      "type": "sse",
      "url": "https://{{ server_ip }}:30502/services/mcp",
      "headers": {
        "Authorization": "Basic YWRtaW46cGlhcC1hZG1pbg=="
      }
    }
  }
}

The MCP server supports RBAC — access is scoped to the authenticated user's Splunk roles. Tools include SPL search, saved searches, lookups, and knowledge object discovery. Learn more
