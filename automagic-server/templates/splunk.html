Splunk — Security Analytics

Collects Tetragon eBPF events, Hubble network flows, and Cilium logs via Fluent Bit and the Splunk HEC.

Splunk receives structured JSON events from the cluster via a Fluent Bit DaemonSet running on every node. Fluent Bit tails the container log files for each selected source and forwards them to the Splunk HTTP Event Collector (HEC) on port 8088.

Three log sources are available: Tetragon (kernel-level ATT&CK events), Hubble (L3/L4 network flows), and Cilium (policy enforcement logs). Toggle them below — the change is applied live without redeploying Splunk.

LOG FORWARDING

Toggle which log sources Fluent Bit forwards to Splunk. Clicking Apply patches the ConfigMap and restarts the DaemonSet pods (~5 s).

Tetragon
Kernel-level ATT&CK events from the export-stdout sidecar. JSON with process context, syscall details, and MITRE technique IDs.
Hubble
L3/L4 network flow logs from the Hubble relay. Shows source/destination pod, port, verdict (forwarded / dropped).
Cilium
Cilium agent INFO/WARN logs. Policy enforcement decisions, endpoint state changes, and BPF map events.
