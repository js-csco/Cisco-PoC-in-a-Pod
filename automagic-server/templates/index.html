SECURE PRIVATE ACCESS

Automate the configuration of your first private access resources and policies to get started with Cisco Secure Access.

Private Resources

Transform your pod into private resources (e.g., SSH server, web server, and more).
Important: After the creation of the resources, please edit each resource and add your Resource Connector to each private resource!

Private Access Policy

This policy enables remote test users to access private resources within your PoC environment.

SECURE INTERNET ACCESS

Automate Access Policies for Secure Internet Access to validate web filtering and isolation scenarios.

Cisco Recommendations

Follow Cisco recommendations for Internet Access rules & decryption of web traffic (MSFT 365 Bypass, Certificate Pinning, Do not Decrypt Destinations).

Internet Access Policies

Generate Internet Access policies for basic use cases: browsing, warning, browser isolation, file type blocking and more. Important: After this step & before testing, there is some manual configuration needed!

Security Profile

Go to Cisco Secure Access Dashboard, create a Security Profile for Internet Access with Decryption enabled and attach it to each Internet Access Policy.

Deploy Cisco Secure Client

☑ To do: Install the Cisco Secure Client on a Test Device.
  1. Download Cisco Secure Client from Secure Access Dashboard > Connect > End User Connectivity > Button "Cisco Secure Client"
  2. Install the following Modules: Core VPN, ZTA, DART, ThousandEyes
  3. Enroll with Test User in ZTA Module
