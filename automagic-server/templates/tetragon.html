ABOUT TETRAGON

Tetragon is a Cilium eBPF-based runtime security engine. Unlike firewalls or network policies that operate at the packet level, Tetragon hooks directly into the Linux kernel — observing and optionally enforcing at the syscall level inside every running container, with near-zero overhead and no code changes required.

⚡ Process Execution

Detect when unexpected binaries run inside pods — shells, curl, wget, nmap.

📂 File Access

Alert on reads of /etc/shadow, service account tokens, and other sensitive paths.

🔴 Kill & Enforce

Optionally terminate offending processes in real time — not just observe.

TRACING POLICIES

A TracingPolicy is a Kubernetes custom resource that tells Tetragon which kernel functions to hook (kprobes) and what to match. When a container triggers a matching syscall — opening a file, executing a binary — Tetragon emits a structured JSON event in real time, visible in the live stream below.

All four policies below are cluster-wide — they cover every pod in every namespace, including the attacker simulation jobs.

WHAT TO EXPECT IN THE STREAM

Tetragon emits four event types, each colour-coded in the stream below:

EXEC — purple

Fired for every binary the attacker pod launches: sh, cat, find, whoami, wget, useradd etc. These are process start events — Tetragon sees every execution across all pods.

KPROBE — red  ⚠ these are the security alerts

Kernel probe events that fired because a TracingPolicy matched a syscall:

  • fd_install — file opened matching /etc/shadow / /etc/passwd
  • fd_install — Kubernetes service account token read
  • security_bprm_check — shell or network tool binary executed
EXIT — grey

Process termination events emitted when each binary finishes. Paired with EXEC events, they give you the complete process lifecycle.
