ABOUT TETRAGON

Tetragon is a Cilium eBPF-based runtime security engine. Unlike antivirus or network policies, Tetragon hooks directly into the Linux kernel — observing and enforcing at the syscall level inside every container.

Process Execution

Detect when unexpected binaries run inside pods — shells, curl, wget.

File Access

Alert on reads of /etc/shadow, /etc/passwd, and other sensitive paths.

Kill & Enforce

Optionally terminate offending processes in real-time — not just observe.

TRACING POLICIES

Deploy both eBPF TracingPolicy objects to the cluster. These run as kernel probes and emit structured JSON events for every matching syscall across all pods.

SIMULATE ATTACK

Launches a short-lived Kubernetes Job in the piap namespace. The simulated attacker pod attempts to read /etc/shadow and /etc/passwd, then spawns a shell — the exact patterns Tetragon's TracingPolicies are watching for.

What the Job executes:
  • cat /etc/shadow
  • cat /etc/passwd
  • sh -c 'id; whoami'
⚠️ Deploy TracingPolicies first so Tetragon can detect the simulation. The Job auto-deletes after 2 minutes.

WHAT TO EXPECT IN THE STREAM

After clicking Simulate Attack, you should see 3 types of events appear, each color-coded:

EXEC — purple

One event per binary launched by the attacker pod: sh, cat, id, whoami. These are process start events — Tetragon sees every execution.

KPROBE — red  ⚠ these are the actual detections

These are the Tetragon security alerts — kernel probes that fired because a policy matched:

  • fd_install — fires when /etc/shadow or /etc/passwd is opened  (policy: detect-sensitive-file-read)
  • security_bprm_check — fires when /sh or /ash is executed  (policy: detect-shell-execution)
EXIT — grey

Process termination events for each of the above, emitted when the process exits.
