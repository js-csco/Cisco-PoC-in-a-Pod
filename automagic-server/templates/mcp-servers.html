MCP Servers

Connect Claude Desktop to this PoC environment via the Model Context Protocol (MCP). MCP lets Claude interact directly with your cluster and browser — no copy-pasting, no screenshots.

WHAT IS MCP?

The Model Context Protocol is an open standard that lets AI models like Claude call tools on external servers in real time. Instead of pasting logs or screenshots into Claude, you configure Claude Desktop with the server URLs below — Claude then queries your cluster, runs browser actions, and reads live data directly during the conversation.

CLAUDE DESKTOP SETUP

Open your claude_desktop_config.json and add the block below. The file is located at:

macOS ~/Library/Application Support/Claude/claude_desktop_config.json
Windows %APPDATA%\Claude\claude_desktop_config.json 
{
  "mcpServers": {
    "kubectl-mcp": {
      "type": "sse",
      "url": "http://YOUR_SERVER_IP:30050/mcp"
    },
    "inspektor-gadget": {
      "type": "sse",
      "url": "http://YOUR_SERVER_IP:30060/mcp"
    },
    "playwright": {
      "type": "sse",
      "url": "http://YOUR_SERVER_IP:30070/mcp"
    }
  }
}

After saving, restart Claude Desktop. The MCP tools will appear in the tool picker.

Kubectl MCP

Port 30050 · Read-only · Official Kubernetes MCP Server

Gives Claude read-only access to the k3s cluster — pods, logs, deployments, services, events, Cilium network policies, and more. Safe to use in demos: no write permissions are granted.

TEST PROMPTS
"List all running pods in the piap namespace with their current status and restart counts."
"Show me the last 30 log lines from the unittcms deployment. Are there any errors?"
"What NodePort services are exposed in the piap namespace? List the port, target, and pod they route to."

Inspektor Gadget MCP

Port 30060 · eBPF observability · Official Inspektor Gadget

Uses eBPF to give Claude live, kernel-level visibility into what your containers are actually doing — DNS queries, TCP connections, syscalls, file access. Ideal for verifying Cisco Secure Access is intercepting traffic, or confirming Tetragon detections.

TEST PROMPTS
"Trace all DNS queries made by pods in the piap namespace for the next 15 seconds. What external domains are they resolving?"
"Which pods in the cluster are making outbound TCP connections right now, and to which IPs and ports?"
"Monitor network traffic from the splunk pod for 20 seconds. Does it make any unexpected outbound connections?"

Playwright MCP

Port 30070 · Headless Chromium · Official Microsoft

Gives Claude a real headless Chromium browser running inside the cluster. Claude can navigate to URLs, take screenshots, fill forms, and read page content — perfect for validating that Cisco Secure Access block, warn, and isolate pages are rendered correctly.

TEST PROMPTS
"Navigate to https://ifconfig.co and tell me the public egress IP shown. Is it a Cisco Secure Access IP?"
"Open https://policy.test.sse.cisco.com and take a screenshot. Does it confirm a Cisco Secure Access policy is active?"
"Try to navigate to https://deepseek.com. Take a screenshot — do you see a Cisco Secure Access block page?"
