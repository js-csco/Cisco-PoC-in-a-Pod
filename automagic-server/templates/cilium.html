ALLOW ALL TRAFFIC

Removes all Cilium network policies from the piap namespace.
Cilium's default behaviour with no policies is allow all — pods can be reached from the local LAN, from the Resource Connector, and from each other without restriction.

What gets allowed:
  • Direct LAN access to all pods (SSH, RDP, web services, Splunk, …)
  • Traffic between pods within the cluster
  • Traffic from the Cisco Resource Connector to all pods

ZERO TRUST APPLICATION ACCESS

Applies a CiliumNetworkPolicy that enforces Zero Trust at the network layer inside the cluster. All workload pods accept ingress only from the Resource Connector — users must go through Cisco Secure Access to reach any service.

Policy: piap-zero-trust — all piap workload pods (nginx, splunk, rdp-server, ssh-server, dashy, sse-check)
  • Ingress: only from the Resource Connector (NODE_IP/32, auto-detected via Downward API)
  • Egress: unrestricted (responses & outbound traffic still flow)
ℹ️ How it works: Restricted services are also switched to externalTrafficPolicy: Local, which disables NodePort SNAT and preserves real client IPs at the pod level. Without this, Cilium would see all NodePort traffic (including direct LAN) as NODE_IP — indistinguishable from connector traffic.
⚠️ Note: Direct LAN access to piap workload pods will be blocked. Hubble UI and the Automagic dashboard remain reachable at all times.

HUBBLE UI

Hubble is Cilium's built-in observability layer. It gives you a real-time visual graph of network flows between pods — which connections are allowed, which are dropped, and by which policy. Open it filtered to the piap namespace to watch traffic while toggling policies above.
