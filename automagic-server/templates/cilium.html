NETWORK POLICY

Allow All Traffic

Removes all Cilium network policies from the piap namespace. Cilium's default with no policies is allow all — pods can be reached from the local LAN, from the Resource Connector, and from each other without restriction.

What gets allowed:
  • Direct LAN access to all pods (SSH, RDP, web services, Splunk, …)
  • Traffic between pods within the cluster
  • Traffic from the Cisco Resource Connector to all pods

Zero Trust Application Access

Applies a CiliumNetworkPolicy that enforces Zero Trust at the network layer. All workload pods accept ingress only from the Resource Connector — users must go through Cisco Secure Access to reach any service.

Policy: piap-zero-trust — all piap workload pods (nginx, splunk, rdp-server, ssh-server, dashy, sse-check)
  • Ingress: only from the Resource Connector Docker bridge (240.0.0.0/29)
  • Egress: unrestricted (responses & outbound traffic still flow)
ℹ️ How it works: Restricted services are switched to externalTrafficPolicy: Local, which preserves real client IPs at the pod level so the Cilium policy can distinguish Resource Connector traffic from direct LAN access.
⚠️ Note: Direct LAN access to piap workload pods will be blocked. Hubble UI and the Automagic dashboard remain reachable at all times.
