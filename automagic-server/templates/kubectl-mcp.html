Kubectl MCP

The Kubectl MCP Server exposes this Kubernetes cluster to any MCP-compatible AI client (Claude Desktop, Claude Code, Cursor, …). Once connected, you can ask questions in natural language — the AI translates them into kubectl-style queries and returns the results.

The server runs in read-only mode: it can describe, list, and inspect any resource in the cluster, but cannot create, modify, or delete anything.

CONNECTION

The MCP server is exposed via Streamable HTTP on a fixed NodePort. Point your Claude client at the URL below — no authentication required inside the PoC network.

http://SERVER_IP:30050/mcp

Claude Desktop

Add the following to your claude_desktop_config.json: 

{
  "mcpServers": {
    "kubectl-mcp": {
      "url": "http://SERVER_IP:30050/mcp"
    }
  }
}

Claude Code (CLI)

Run once in your terminal to register the MCP server: 

claude mcp add --transport http kubectl-mcp http://SERVER_IP:30050/mcp

WHAT YOU CAN ASK

Once connected, try questions like these in your Claude client:

  • What pods are running in the piap namespace?
  • Show me the Cilium network policies in the cluster.
  • What resources does the automagic deployment request?
  • Are there any pods in a non-Running state?
  • What services are exposed as NodePort?
  • Explain the RBAC setup for the automagic-sa service account.
