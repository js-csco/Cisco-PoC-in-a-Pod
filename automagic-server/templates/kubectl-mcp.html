Kubectl MCP

What is MCP?

The Model Context Protocol (MCP) is an open standard developed by Anthropic that allows AI models like Claude to securely call external tools and services. Instead of just answering questions from training data, Claude can use MCP servers to take real-time actions — querying APIs, running commands, reading files, and more.

MCP follows a client-server architecture. Your AI client (e.g. Claude Desktop or Claude Code) connects to one or more MCP servers. Each server exposes a set of tools the model can call — similar to plugins, but standardised and transport-agnostic (stdio, SSE, or Streamable HTTP).

Kubectl MCP Server

The Kubectl MCP Server running in this cluster gives Claude direct, read-only access to your Kubernetes environment. You can ask Claude questions like "Which pods are not running?" or "Show me the logs of the automagic pod" and it will query the cluster in real time to answer. It exposes standard Kubernetes operations as MCP tools — no kubectl CLI needed on your machine.
