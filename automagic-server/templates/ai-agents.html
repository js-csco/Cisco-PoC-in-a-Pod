AI Agents — DefenseClaw

Enterprise security governance for agentic AI — powered by Cisco DefenseClaw and OpenClaw.

AI agents are powerful. That’s exactly what makes them dangerous.

An LLM-powered agent can call tools, execute code, browse the web, and talk to APIs. Without guardrails, a compromised skill, a prompt-injection attack, or a hallucinated shell command can turn your helpful assistant into a security incident.

DefenseClaw is Cisco’s open-source answer to this problem. It sits between the AI agent and the outside world — scanning every skill before it loads, inspecting every tool call before it executes, and blocking anything that smells like secrets leakage, injection, or command-and-control.

OpenClaw is the LLM agent framework that DefenseClaw protects. Think of OpenClaw as the brain (the agent that reasons and acts) and DefenseClaw as the immune system (the gateway that enforces policy, audits behaviour, and stops threats).

Together they give you an AI agent that can actually do useful work in a production environment — without keeping your security team up at night.

ARCHITECTURE

How DefenseClaw, OpenClaw, and your existing k3s services fit together.

OpenClaw Agent

LLM-powered agent that reasons, plans, and calls tools. Hosts the DefenseClaw plugin which intercepts every tool call via a before_tool_call hook.

DefenseClaw Gateway

Go-based policy engine. Runs CodeGuard static analysis, skill/MCP scanning, tool inspection, and a guardrail LLM proxy. Logs every decision to SQLite and forwards audit events to Splunk via HEC.

CLI Dashboard

Python TUI for operators. Trigger scans, review audit logs, manage policies, and watch agent activity in real time — all from the terminal.
